# Crypto Monitoring Service (MIPT Project)

Сервис для мониторинга котировок криптовалют.
Состоит из:

- **сервера на Go** (Chi + SQLite)
- **агента-сборщика данных**

---

# 🚀 Запуск на локальном сервере

## 1. Подготовка окружения

Убедись, что у тебя установлено:

- **Go 1.21+**
- **SQLite**

---

## 2. Сборка и запуск сервера

Сервер слушает порт **`:8080`** и сохраняет данные в файл **`metrics.db`**.

```bash
# Перейди в корень проекта
go mod tidy
go run cmd/main.go
```

---

# 🛠 Обновление окружения и Swagger

Если ты меняешь структуру данных
(например, добавил новое поле в `models.Stat`), нужно:

1. обновить **Swagger-документацию**
2. пересоздать **схему базы данных**

---

## Обновление Swagger

```bash
swag init -g cmd/main.go
```

---

## Сброс базы данных

Если изменились колонки:

1. Удали файл `metrics.db` в корне проекта
2. Перезапусти сервер

Сервер автоматически создаст новую базу с актуальной структурой.

---

# 📖 Использование Swagger UI

Интерфейс доступен по адресу:

```
http://localhost:8080/swagger/index.html
```

---

## Тестирование `POST /update`

1. Открой вкладку **POST /update**
2. Нажми **Try it out**
3. В поле `body` укажи JSON

Поле `timedump` можно оставить пустым — сервер заполнит его автоматически.

```json
{
  "symbol": "BTCUSDT",
  "price": "67500.50",
  "source": "manual_test"
}
```

4. Нажми **Execute**

### Ожидаемый результат

```
Code: 200
или
Code: 201
```

Если появляется **500**, проверь **логи сервера** — там будет ошибка SQL.

---

# 🔎 Проверка статуса (Ping)

Открой в браузере:

```
http://localhost:8080/ping
```

Если отображается

```
pong
```

значит:

- сервер работает
- роутер настроен корректно

---

# 🔍 Где искать данные?

## Логи

Выводятся прямо в **терминале сервера**.

Обращай внимание на строки:

- `Received stat`
- `DATABASE SAVE ERROR`

---

## База данных

Данные сохраняются в файл:

```
metrics.db
```

Открыть базу можно через:

**SQLite Viewer** в **VS Code**.
